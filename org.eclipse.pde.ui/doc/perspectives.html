Perspectives

Identifier

org.eclipse.ui.perspective

Description

This extension point is used to add layout engines to the workbench. A layout engine is used to define a layout for a perspective. The user can select a layout by invoking the "Switch To" submenu of the Perspective menu.

Markup

   <!ELEMENT extension (layout*)>
   <!ATTLIST extension
     point CDATA #REQUIRED
     id    CDATA #IMPLIED
     name  CDATA #IMPLIED
   >

  • point - a unique extension point identifier
  • id - an optional extension identifier
  • name - an optional extension name

   <!ELEMENT description (#CDATA)>

   <!ELEMENT layout (description?)>
   <!ATTLIST layout
     id    CDATA #REQUIRED
     name  CDATA #REQUIRED
     class CDATA #REQUIRED
   >

  • id - a unique name that will be used to identify this layout.
  • name - a translatable name that will be used in the workbench window menu bar to represent this layout.
  • class - a fully qualified name of the class that implements org.eclipse.ui.ILayoutEngine interface.

Example

The following is an example of a layout extension: 

   <extension 
        point="org.eclipse.ui.layout"> 
        <layout 
            id="org.eclipse.ui.defaultLayout" 
            name="Default Layout" 
            class="org.eclipse.ui.internal.DefaultLayout"> 
        </layout> 
    </extension>

API Information

The value of the class attribute must be the fully qualified name of a class that implements org.eclipse.ui.ILayoutEngine. The class must supply the initial layout for a perspective when asked by the workbench.

Supplied Implementation

The workbench provides a default layout. Additional layouts may be added by plug-ins. They are selected using the "Switch To" submenu of the View menu.
